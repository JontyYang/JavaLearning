1.在一个方法中如果参数是字符数组，参数值是字符数组的引用，因此外部数组值变了，内部值也变（引用相同);但是对于字符串的创建，如果参数是字符数组，则参数的值是
字符数组的复制，外部改变，内部并不变，保证了字符串的不变性。
2.Java中String和char在内存中始终以Unicode进行编码
3.数据传输时使用UTF-8进行编码
4.使用ArrayList时，如果不定义泛型类型时，泛型类型实际上就是Object
5.Arrays.sort(Object[])可以对任意数组进行排序，但待排序的元素必须实现Comparable<T>这个泛型接口
6.对于泛型<T>而言，尽管在定义时对T进行类型指定，但是对于编译器而言它会使用擦拭法自动忽略T类型默认是Object类型，因此虚拟机看到的是Object类型，
因此我们获取类形时获取得到的是只是类名（不包含T的具体类型），因此该泛型类中不能实例化T类型（new T（）），会使用擦除法为Object类型，这样各种类型的T
还是Object就没有了区别，显然编译器要阻止这种问题的发生。
7.如果一个类继承了泛型类，那么他是继承了泛型类的具体类型的。
8.数组初始化后大小不可变，只能按索引存取。
9.如果放入的元素需要使用List（或者其他集合类）的contains，indexof,remove方法（参数为元素），那么放入的元素就需要实现equals方法

17.SortedSet继承了Set接口，TreeSet是SortedSet实现类，该类可以实现元素的排序，添加的元素必须正确实现Comparable接口。
   map也是同样的，它也有SortedMap接口，TreeMap是实现类，该类实现按Key实现map的排序，添加的key必须实现Comparable接口。
18.使用String、Integer等类型的时候，这些类型都是不变类，一个不变类具有以下特点：
   定义class时使用final，无法派生子类；
   每个字段使用final，保证创建实例后无法修改任何字段。
   为了保证不变类的比较，还需要正确覆写equals()和hashCode()方法，这样才能在集合类中正常使用。
19.JVM存在主内存，线程有自己的工作内存，对于共享变量需要写入主内存。
20.join阻塞的是调用该方法线程所在线程，并不是所有线程。
